NFT metadata allows you to attach rich content to prediction market assets and Prediction Market Organizations. This feature supports a wide range of media formats and hosting solutions.

## Supported Media Formats

### Images
PNG, JPEG, GIF, WEBP, TIFF, BMP

### Audio
MP3, MP4, M4A, OGG, FLAC, WAV, WMA, AAC

### Video
WEBM, MOV, QuickTime, AVI, WMV, MPEG

### Documents
PDF, DOCX, ODT, XLSX, ODS, PPTX, TXT

### 3D Models
OBJ, FBX, glTF, 3DS, STL, COLLADA, 3MF, Blend, SketchUp, VOX

## Metadata Fields

When creating or editing a prediction market or PMO, you can attach the following NFT metadata:

- **Title** — Display name for the content
- **Artist/Creator** — The creator's name or identity
- **Narrative** — Extended description or story
- **Tags** — Comma-separated tags for categorization and search
- **Type** — The type of content (text, image, video, audio, 3D, etc.)
- **IPFS media link** — The IPFS CID or URL for the media file

## IPFS Hosting

NFT media is stored on IPFS (InterPlanetary File System). Several hosting services make it easy to upload and pin your files:

- **Pinata** — Popular IPFS pinning service
- **NFT.storage** — Free IPFS storage for NFTs
- **Web3.storage** — Decentralized storage solution
- **Fleek** — IPFS hosting with easy deployment
- **Infura** — IPFS API access
- **StorJ** — Decentralized cloud storage
- **Eternum** — Blockchain-focused storage

## IPFS Gateway

The UI uses an IPFS gateway to display media content. The default gateway is `https://ipfs.io/ipfs/`. You can configure a custom gateway in [Visual Settings](/docs/customizing-visuals.html).

## When to Use NFT Metadata

- **Prediction markets** — Attach a visual representation or documentation for your prediction
- **Organizations** — Add a logo, brand assets, or organizational documents to your PMO
